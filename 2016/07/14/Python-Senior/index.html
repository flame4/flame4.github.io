1. With语句

With语句在Python2.6中被加入标准，作为try/finally的替代，用于资源访问进行控制的场合。

先要明白几个概念：

1. 上下文管理器： 特殊对象，实现了__enter__()和__exit()__方法的对象，只有这种对象才能够支持with语句，其定义了with在执行时候要建立的"运行时上下文"，负责执行进入和退出操作。
2. 运行时上下文： 在语句体执行前通过__enter__()进入运行时上下文，语句体执行完后通过__exit()__退出。简单的理解就是，with的语句体执行之前需要做的一些预处理工作和执行完之后的擦屁股操作，比如打开文件什么的。
3. 上下文表达式： 我们可以自己定义一个上下文管理器给with使用，但一般为了避免麻烦，可以使用with自动生成上下文管理器。这通过调用with的时候送给其一个上下文表达式完成，可以在后面更好的理解。
4. 语句体： with包裹的要执行的代码块。

5. python自省(反射)

自省一定程度上破坏了对象的私有特性, 并提供了注入的可能性. 反射的好处是可以增加程序的灵活性, 让很多东西可以在运行时才确定, 而不是直接写死在代码里. 一句话说, 就是可以方便的完成开闭原则, 让很多将来可能修改的东西不至于写死在代码里.

比如下面的继承关系: A为父类工厂, B, C, D, E … 为子类, 有的子类存在方法AA(), 有的没有,那么如果我们一个函数要执行, 没有反射我们只能对于每一类写判断, 而且还有可能不断修改, 而如果有了反射, 我们只需要判断这个类有没有AA方法即可, 对于逻辑不变的需求, 这里永远不改写.

python中虽然可以传递函数作为参数, 但是函数也是作为一个对象存在的.对于所有的对象, python的几个内建方法即可完成反射,十分的简单.

  1. dir() 获取对象属性列表
  2. hasattr() 是否存在属性
  3. getattr() setattr() 设置, 查询属性.

查看所有属性列表.列出几个常用的类内置属性:

  1. iter: generator的属性, 可以调用, 所以生成器内部其实还是迭代器.
  2. str: 重载print,结构化输出
  3. len: 重载len
  4. getitem: 重载[]
  5. ge: 重载比较运算

6. python lambda表达式与函数式编程

首先看编程范式, 编程范式(用心阁的回答)可以认为是程序员去认识自己的程序的方式, 你怎么样去思考你的程序, 角度决定你的程序是哪种情况. 常见的编程范式有下面几种:

  1. 命令式: 强调人和程序思维的一致性, 简单说就是图灵机模型. 数据在给定的过程中流动, 并最终形成理想的输出. 人需要告诉机器, 对于来的数据要怎么处理, 每一步应该做什么.
  2. 逻辑式: 神牛语言, 大概用在逻辑演算, 语言领域比较强, 不感兴趣就不深挖了.
  3. 函数式: 函数是一段程序中, 唯一的元素. 很多关于函数式的资料上总结写了对于函数式语言最重要的几个特性: 变量不变性, 依赖递归, 高阶函数, 无状态. 一句话来说, 函数式编程是聪明人玩的游戏. 命令式语言是人用人的语言来思考, 并且告诉机器怎么做, 而函数式语言则你要先对你做的事情用函数的方式重新思考并且给出定义, 然后就不管了. 相比于告诉计算机怎么做, 你要用计算机可以直接理解的方式告诉它应该干什么.比如下面这个例子:

==========================我是分割线=======================

对于现实生活的抽象, 使用面向对象的思想无疑是最为让人们接受的, 因为这种思维模拟了现实生活的关系. 面向对象不是一种编程范式, 其更像是几种语言范式的组合, 但核心还是命令式的. 这本身是正确的, 因为这样可以以更小的代价构建应用. 但在一些特定的领域, 传统的思维模式会造成很多的麻烦, 这时候一些新的编程范式却可以很简单的解决问题.

在并行化的领域, 函数式的一些思维就有天然优势. 无状态的函数表示着不需要加锁(因为一个变量只能是只读的), 那么很多问题就被规避掉了. Golang的goroutinue就是这个道理, 也许是因为这些原因, Golang也是重新思考了编程范式, 所以才会在函数传值的时候, 把struct等一些似乎为指针传递的类型定义为值传递吧. 但这里的并行化和不变性也只是借助了函数式的思想, 并不是纯粹的函数式. 在实际过程中, 还需要解决的是多个并行程序之间的同步问题和共享内存问题.

python中的函数式编程思想体现在:

  1. 匿名函数, 函数闭包, 高阶函数的实现.
  2. 不可变数据类型(string, tuple, int等)
  3. 对一个函数无状态的支持. 使用内置函数封装for, while循环, 尽可能消除中间过程中表示状态的临时变量, 使得函数维持清晰的无状态.

对于闭包来说, 函数闭包就类似于在函数体内又定义了一个局部函数, 可以使用这个函数体内的变量.类似于普通的模块函数和模块中定义的全局变量的关系：修改外部变量能影响内部作用域中的值，而在内部作用域中定义同名变量则将遮蔽（隐藏）外部变量.但闭包对于函数内部变量的引用,也可能导致外部变量的变化, 比如第二段程序, 是不被建议这么写的.

对于内建函数, python的函数式编程为了消除for循环产生的中间变量, 将这些敏感的过程都封装成了内建函数以提供函数式编程特性, 因为for循环针对一个列表, 所以就替换掉了常见的操作:

  1. map(function, iterable) 对list的每个元素作用上function.
  2. reduce(function, iterable) 对list的元素之间使用function, 使得所有元素变为一个元素
  3. filter(function, iterable) 对list的每个元素做function的过滤.

最后, 针对”做什么”和”怎么做”的想法, 因为函数式强调做什么(比如对这个列表的所有元素乘2), 而不强调怎么做. 但对于命令式的思维而言, 想要知道做什么, 本质上还是得思考怎么做, 因为只有知道怎么做, 才能知道做什么. 但这样就还是跳不出命令式的思维. 那么我觉得, 想跳出命令式的思维, 就不应该想怎么做, 而应该想, 这个事情做完后的结果是什么, 它和原来的对应关系是什么. 想明白后, 你只需要把这个对应关系用函数式语言的工具(递归, 内建函数)描述清楚就好了. 具体细节还是要真的写过很多之后才能理解把.
